# 页面要求

## 基础布局

1. 要求页面能够在PC端与移动端能够自适应展示数据
2. 页面元素水平居中, 由于PC端屏幕宽度大，主要元素在屏幕中间展示, 界面两边应该留出足够空白

## 页面风格

1. 整体风格应该简约, 色系以黑白灰为主
2. 字体选择一种美观字体
3. 颜色支持明亮模式与黑夜模式


## 图表要求

### calendar 图表

1. 仿照 github commit calendar 图表, 将单日数据绘制成 calendar 图表, 横轴表示月份, 纵轴表示星期几. 每个单元格通过颜色深浅表示不同的跑步距离。当鼠标悬浮在单元格时, 应当展示当天跑步的具体数据细节: 跑步距离, 平均配速, 平均心率, 跑步地点, 开始时间。 如果当天没有跑步，则不展示细节
2. 由于有很多年的跑步数据, 所以一次性只展示最近一年的 calendar, 提供按钮能够进行年份翻页

### 分布图
